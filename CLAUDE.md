# Vinctum Core

Decentralized data courier platform built with Go microservices, gRPC, and libp2p.

## Architecture

5 microservices communicating over gRPC, with a REST gateway for external clients:

| Service     | Port  | Role                                      |
|-------------|-------|-------------------------------------------|
| Identity    | 50051 | Auth (JWT + bcrypt), user management      |
| Discovery   | 50052 | P2P peer registry via libp2p/Kademlia DHT |
| Routing     | 50053 | Route computation, relay management       |
| Transfer    | 50054 | Chunk-based encrypted file transfer       |
| Gateway     | 8080  | HTTP-to-gRPC proxy for browser clients    |

Backing stores: PostgreSQL 16, Redis 7 (token blacklist).

## Project Layout

```
cmd/                  # Service entry points (one main.go per service)
proto/                # Protobuf definitions (buf.build toolchain)
services/             # Business logic: handler/ + repository/ per service
  transfer/storage/   # File-based chunk store
internal/             # Auth (JWT, blacklist), migrator, p2p node
pkg/                  # Shared: config (viper), logger (zerolog), crypto, middleware
scripts/migrations/   # SQL schema files (embedded, auto-applied on boot)
config/               # YAML configs (config.dev.yaml)
deployments/docker/   # Dockerfiles + docker-compose.yml
doc/                  # Threat model, 14-week project plan
```

## Code Generation

Two codegen pipelines -- always run after changing `.proto` or `.sql` files:

```bash
make generate          # runs both below
make generate-proto    # buf generate (proto → Go stubs)
make generate-sql      # sqlc generate (SQL queries → type-safe Go)
```

Config: `buf.yaml` + `buf.gen.yaml` for proto, `sqlc.yaml` for SQL (4 configs, one per DB-backed service).

## Build & Run

```bash
make build             # compiles all 5 services to bin/
make run-identity      # go run ./cmd/identity/...
make run-discovery
make run-routing
make run-transfer
make run-gateway
make docker-up         # docker compose up --build -d
make docker-down
```

## Testing

```bash
make test              # go test ./... -v -race -count=1
make test-cover        # generates coverage.out + HTML report
make lint              # golangci-lint run ./...
```

CI (`.github/workflows/ci.yml`) runs lint -> test (with Postgres + Redis) -> build matrix -> Docker push on main.

## Key Conventions

- **Commit messages**: conventional commits with scope -- `feat(transfer):`, `test(routing):`, `chore(infra):`, etc.
- **Repository layer**: generated by sqlc, never hand-edited. Each service has a `Querier` interface for testability.
- **Handler tests**: use `fakeQuerier` implementations, no real DB needed.
- **Config**: Viper with `VINCTUM_` env prefix. Nested keys use `_` separator (e.g., `VINCTUM_AUTH_JWT_SECRET`).
- **Migrations**: embedded SQL files in `scripts/migrations/`, auto-applied on service startup via `internal/migrator`.
- **gRPC auth**: `pkg/middleware` interceptors validate JWT on all RPCs except Register/Login/RefreshToken.
- **Proto imports**: `buf.build/googleapis/googleapis` is a dependency.

## Security Notes

- JWT uses HMAC-SHA256 (not RSA) -- secret must be strong in prod.
- Discovery service currently has **no auth interceptor** -- known gap.
- TLS is supported but disabled in dev (`tls_enabled: false`).
- See `doc/threat_model.md` for full STRIDE analysis and open items.
